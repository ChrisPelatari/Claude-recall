**Note on duplicated entitlements files:** Three copies of `AIMemoryReader.entitlements` exist on disk:
1. `/AIMemoryReader.entitlements` (repo root) — referenced by `project.yml` as `CODE_SIGN_ENTITLEMENTS`
2. `/AIMemoryReader/AIMemoryReader.entitlements`
3. `/AIMemoryReader/Sources/Resources/AIMemoryReader.entitlements`

Only the repo-root copy is consumed by the build. Treat the other two as stale duplicates; do not edit them and expect the change to take effect. When editing entitlements, edit `/AIMemoryReader.entitlements` only.

## Directory Purposes

**`AIMemoryReader/Sources/App/`:**
- Purpose: Entry points and lifecycle wiring
- Contains: `@main` Scene definition, NSApplicationDelegate, top-level `Notification.Name` extensions
- Key files: `AIMemoryReaderApp.swift`

**`AIMemoryReader/Sources/Models/`:**
- Purpose: State container and value/reference types that represent the app's data model
- Contains: `AppState` (state hub), `AISource` (catalog + auto-detection), `FileNode` (tree node), `SettingsStore` (persistence), `AppTheme` / `ThemePalette`
- Key files: `AppState.swift` (491 lines — the largest file in the project and the canonical place to look for state behavior)

**`AIMemoryReader/Sources/Views/`:**
- Purpose: SwiftUI presentation, `NSViewRepresentable`/`UIViewControllerRepresentable` adapters
- Contains: Top-level platform routers, sidebar, detail/markdown/find/edit/TOC views, banners
- Key files: `ContentView.swift` (platform routing entry), `DetailView.swift` (largest view, 626 lines, hosts find + TOC + edit)

**`AIMemoryReader/Sources/Utilities/`:**
- Purpose: Stateless services and OS-API adapters
- Contains: File I/O, search, theming, syntax highlighting, PDF export, update checker, sandbox bookmarks, window chrome, image provider, find-mode renderer
- Key files: `BookmarkStore.swift` (sandbox gate), `UpdateChecker.swift` (only network code), `FileWatcher.swift` (FSEvents)

**`AIMemoryReader/Sources/Resources/`:**
- Purpose: Non-Swift bundle resources
- Contains: `Info.plist`, `PrivacyInfo.xcprivacy`, asset catalog, entitlement copies
- Key files: `Info.plist` (URL scheme + document types), `PrivacyInfo.xcprivacy` (privacy manifest)

**`docs/`:**
- Purpose: GitHub Pages content
- Contains: `index.html`, `privacy.html`, `claude-code-memory-guide.html`, generated `codebase/` site
- Not bundled into the app.

**Root-level docs (markdown, not source):**
- `CLAUDE.md` — primary orientation doc for Claude Code instances
- `MAS_METADATA.md` — submission steps, archive command, App Store nutrition label
- `LICENSING.md` — dual distribution rationale
- `CLAUDE_CLOUD_MEMORY_SPEC.md` — unimplemented forward-looking design
- `PLAN.md`, `V2-PLAN.md`, `V3-PLAN.md` — completed-work changelogs, treat as historical context only

## Naming Conventions

**Files:**
- Swift sources: `UpperCamelCase.swift`, one primary type per file matching the filename (`AppState.swift` → `AppState`)
- Cross-platform shared file: keep the platform-neutral name and use `#if os(...)` inside
- macOS-only file: still in the shared `Sources/` tree, but the entire content wrapped in `#if os(macOS) … #endif` (e.g. `PDFExporter.swift`, `WindowChrome.swift`)

**Directories:**
- Singular role names: `App`, `Models`, `Views`, `Utilities`, `Resources` (no plural forms beyond these)
- All grouped under `AIMemoryReader/Sources/`

**Types:**
- `UpperCamelCase` for types
- Enum cases: `lowerCamelCase`
- Singletons exposed as `static let shared` (`SettingsStore.shared`, `BookmarkStore.shared`, `UpdateChecker.shared`)
- `Notification.Name` extensions defined at file top with `editorManualSave`, `exportPDF`, `openFileFromSystem`, `fileWatcherDidDetectChange`, `settingsDidSyncFromiCloud`, `editorShowFindBar`

**Bundle identifiers:**
- macOS: `com.aitools.ai-memory-reader`
- iOS: `com.aitools.ai-memory-reader-ios`
- URL scheme: `aimemoryreader://` (lowercase, no hyphens)

## Where to Add New Code

**New SwiftUI view (cross-platform or macOS-specific):**
- File location: `AIMemoryReader/Sources/Views/`
- For macOS-only: wrap the whole file in `#if os(macOS) … #endif`
- Inject `AppState` via `@Environment(AppState.self) private var appState`
- Read theme via `@Environment(\.themePalette) private var palette`; branch on `palette.isEyeCare` for Solarized colors
- If you need a platform router pattern, see `Views/ContentView.swift` for the canonical `#if os(macOS) … #else … #endif` split

**New utility / service:**
- File location: `AIMemoryReader/Sources/Utilities/`
- If it touches AppKit: wrap the entire file in `#if os(macOS)`
- If it's a process-level singleton: expose `static let shared = ...` and mark `@unchecked Sendable` only if you handle thread safety (see `SettingsStore` and `BookmarkStore` for the pattern); prefer `@MainActor` for UI-touching singletons (`UpdateChecker`)
- Add no `print` debugging without removing it — there is no log framework

**New persisted preference:**
- Add a key to `SettingsStore.Key` (`AIMemoryReader/Sources/Models/SettingsStore.swift` line 14)
- Expose a typed accessor on `SettingsStore` (mirror the existing `recentFolders` / `appThemeRaw` pattern, lines 40–63)
- Add to `migrateLocalToiCloudIfNeeded()`'s `keysToMigrate` array (line 121) so existing users' values are pushed to iCloud
- Add a property to `AppState` with a `didSet` that writes through to `SettingsStore.shared` (mirror `appTheme` lines 33–37)

**New AI source (auto-detected agent):**
- Add an entry to `AISource.allSources` in `AIMemoryReader/Sources/Models/AISource.swift` line 71
- Use a relative `path:` from the home directory (e.g. `.gemini`); the `url` computed property resolves it
- Pick an SF Symbol via `iconName:` (existing sources use `terminal`)
- The new agent is auto-detected on launch via `AISource.detectAvailable()`

**New markdown rendering capability:**
- Theming: extend `MarkdownUI.Theme.memoryReader(palette:)` in `AIMemoryReader/Sources/Utilities/MemoryReaderTheme.swift`
- Code highlighting for a new language: add the keyword set to `basicHighlight(_:language:)` in `AIMemoryReader/Sources/Utilities/SplashCodeSyntaxHighlighter.swift` line 35
- Find-mode (NSAttributedString) rendering: extend `ReadableMarkdownRenderer.applyBlockStyles` / `applyInlineStyles` in `AIMemoryReader/Sources/Utilities/ReadableMarkdownRenderer.swift`

**New URL-scheme action:**
- Add a branch in `AppState.handleURL(_:)` (`AIMemoryReader/Sources/Models/AppState.swift` line 265). Keep the path-traversal and absolute-path guards for any new path-bearing query parameter.
- If the action also needs CLI access, extend the case block in the `aimr` script at the repo root.

**New macOS menu command:**
- Add to the `.commands` block in `AIMemoryReaderApp.body` (`AIMemoryReader/Sources/App/AIMemoryReaderApp.swift` lines 89–183). Existing groups: replacing `.newItem`, after `.textEditing`, after `.toolbar`, replacing `.saveItem`, after `.sidebar`, replacing `.help`.

**New file-type support (beyond `.md` / `.json`):**
- Add the extension to `FileNode.supportedExtensions` in `AIMemoryReader/Sources/Models/FileNode.swift` line 12
- If it needs special rendering, branch in `MarkdownDetailView` / `iOSDetailView` (`Views/DetailView.swift`, `Views/ContentView.swift`)
- Add the UTI to the `NSOpenPanel.allowedContentTypes` in `AppState.openFolder()` and to `DocumentPickerView`'s `types` array
- Add a `CFBundleDocumentTypes` entry in `Info.plist` if it should be openable from Finder
